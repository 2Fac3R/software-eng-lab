# Domain 2: Design High-Performing Architectures

Esta sección trata sobre cómo optimizar el rendimiento de tus recursos en AWS.

## Compute Performance
*   **EC2 Instance Types:** Seleccionar la familia adecuada (C para cómputo, R para memoria).
*   **Lambda:** Cómputo sin servidor que escala automáticamente.

## Storage Performance
*   **EBS (Elastic Block Store):** Uso de volúmenes gp3 e io2 para máximo rendimiento.
*   **EFS (Elastic File System):** Almacenamiento de archivos compartido para Linux.

## Database Performance
*   **ElastiCache:** Caché en memoria (Redis/Memcached).
*   **DynamoDB Accelerator (DAX):** Microsegundos de latencia para DynamoDB.
