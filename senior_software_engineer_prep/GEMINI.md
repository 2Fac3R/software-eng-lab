# Project Context: Senior Software Engineer (Critical Infrastructure)

## 1. Project Overview
This project is a comprehensive training plan for a **Senior Software Engineer** role focused on building critical financial infrastructure. The plan is based on a job description emphasizing robust architecture, deep database knowledge, and modern Python practices.

## 2. Key Technologies & Concepts
- **Core Language:** Advanced, idiomatic Python 3.x with a focus on static typing (`PEP 484`).
- **Quality Tooling:** `Pydantic` for data validation, `ruff` for linting/formatting, `mypy` for type checking, and `uv` for environment management.
- **Architecture:** Domain-Driven Design (DDD) and Clean Architecture.
- **Database:** Deep expertise in PostgreSQL >= 16, including partitioning, `JSONB`, advanced indexing (GIN), RLS, and `PL/PgSQL`.
- **Testing:** Test-Driven Development (TDD) and Behavior-Driven Development (BDD) using `pytest`.
- **CI/CD:** Automated pipelines using tools like GitHub Actions.
- **Frameworks:** Django and FastAPI, used as the outer layer in a Clean Architecture.
- **Deployment:** Docker and Kubernetes (as a "nice-to-have" skill).

## 3. Module Structure
The training plan is divided into 6 modules, each with its own directory containing theory, challenges, solutions, cheatsheets, flashcards, and exams.

*   **`01_advanced_python_and_quality_ecosystem/`**: Focuses on modern Python and its quality toolchain.
*   **`02_advanced_software_architecture_ddd_and_clean_architecture/`**: Covers DDD and Clean Architecture principles.
*   **`03_postgresql_expertise/`**: A deep dive into advanced PostgreSQL features.
*   **`04_quality_testing_and_cicd/`**: Details TDD, BDD, advanced pytest, and CI/CD automation.
*   **`05_web_frameworks_django_and_fastapi/`**: Explains how to use web frameworks within a Clean Architecture.
*   **`06_containerization_and_deployment/`**: Covers Docker and Kubernetes fundamentals.
